## 第二次重點：Text-Based Art 視覺體驗

這次特別專注在 ASCII 世界本身的視覺設計，走得比第一次慢，刻意在每個關卡多停一下觀察畫面。

---

## 視覺系統整體印象

整個世界用 `RichTextLabel` + BBCode 顏色標記渲染，Consolas 28px。
第一眼看起來很極簡，但盯久了就會發現顏色系統做了非常精確的分層：

| 元素 | 顏色 | 感受 |
|---|---|---|
| 牆壁 `#` `I` | `#EEEEEE` 白 | 硬、冷、制度感 |
| 地板 `.` `,` | `#252525` 深灰 | 幾乎消失，只剩輪廓 |
| 廢料區 `%` | `#4A3A1A` 暗棕 | 腐敗感，城市底層 |
| 門/出口 `>` | `#CC8833` 琥珀 | 唯一暖色，誘惑感 |
| 玩家 `@` | `#DDCCAA` 米白 | 存在感但不搶眼 |
| Glitch | `#FF1111` 紅 | 突然、不受控 |

## 各場景視覺紀錄

### 地鐵站 `Station.tscn`

```
[ 虎寨城地鐵站 //  LINE 3  //  02:47  //  NO SERVICE TONIGHT ]
```

地圖設計最精簡的一個場景。月台區域用 `.` 圍出來，軌道那排 `_ _ _ _ _` 讓人立刻理解這是鐵路，不需要說明文字。

`%` 散落在月台邊緣（L12 `%     %  %    %`）——廢棄感，也是 glitch 的來源點之一。

老鼠 NPC 顯示為 `?`，配色是 `#66CC66` 綠。放在昏暗地圖裡有一種「這個東西不太對勁」的視覺預警，跟角色本人的氣質完全吻合。

---

### 警察局辦公室 `Office.tscn`

```
[ 虎寨城警察局  //  辦公室  //  08:55 ]
```

`|` 垂直線在頂部密集排列（L6 `|  |  |  |  ...`），像鐵欄杆或日光燈管，壓迫感很強。

`[DESK]`、`[FILING]` 這些文字標籤直接寫進地圖字串，土法煉鋼但非常有效——一眼就知道功能。

李先生 `L` 的顏色是 `#E6BF33` 金黃，官僚的油膩感用顏色表現出來了。

底部 `% %  % %%` 那排廢料區在辦公室裡出現，感覺特別諷刺：精良整齊的官僚體制，底下是一樣的腐爛。

---

### 街道 `Street.tscn`

```
[ 虎寨城  //  中環路  //  深夜 ]
```

建築用 `|  BLDG  |` 表示，城市輪廓一目了然。

門口的 `>` 在最右側（`48,13`），顏色 `#CC8833` 在一片灰暗地圖裡非常顯眼——玩家很自然就會往那個方向走，不需要任何提示文字。

地上的 `? ?` 散布（L9、L11）——遊民？垃圾？故意曖昧，讓人自己填入意義。

## Glitch 系統觀察

`stress_level` 決定 glitch 頻率和數量：

- `stress_level = 0`：每 2.5 秒觸發一次，只換 0~2 個字符
- `stress_level` 拉高：間隔縮短，count 增加，甚至開始腐蝕地板 `.`

三個 glitch pool：
- 牆壁 `#` `I` → `l 1 ! i ¦ ╎ ;`（形近字，微妙失真）
- 廢料 `%` → `@ ! $ Ψ Ω Δ ░ ▓ × †`（混亂符號）
- 地板 `.`（高壓時才觸發）→ `v ' \` · ; ,`（細微腐蝕）

第二次玩特別注意：牆壁 glitch 用的是形近字（`l`、`1`、`!`），不是完全亂掉，而是讓你懷疑「那個字還是原來那個字嗎？」。這個設計決策很聰明，比直接換成完全不相干的符號更令人不安。

## 玩家 `@` 的存在感

`@` 在 ASCII 世界裡是最傳統的「玩家符號」，米白色 `#DDCCAA` 讓它在任何背景上都可辨識，但又不過於突出。

它在地圖裡就是一個字符，跟所有其他字符共存，沒有任何動畫或特效——這種「存在感低」其實和皮耶爾這個角色的設定完全一致：「那是張再平庸不過的日耳曼人面容……過了三十秒，整張臉就在記憶裡模糊了起來。」
