## 議會工作室

### 起源

發現 [Claude Code Game Studios](https://github.com/Donchitos/Claude-Code-Game-Studios)——把 Claude Code 組織成正式工作室的模板。49 個 agent、73 個 slash command。分析後決定不照抄，建輕量版本。

### 設計原則

**零指令記憶。** 不需要背任何 command。有需要才打，不打不消耗 token。

### 實作

四個 slash command 存在 `.claude/commands/`：

| Command | 功能 |
|---------|------|
| `/story` | 派敘事議員——NPC 文字、故事語氣、Markdown 格式 |
| `/code` | 派程式議員——GDScript、Godot 系統、已知 bug |
| `/qa` | 派測試議員——GDUnit4、回歸分析 |
| `/council` | 三個同時並行，最後合併 |

這些檔案只在輸入 `/` 時載入，平時不佔 context。

## 技術備忘

### GDUnit4 v6.1.3 assertion 坑

| 錯誤寫法 | 正確寫法 |
|---------|---------|
| `assert_dict(d).contains_key(k)` | `assert_bool(d.has(k)).is_true()` |
| `assert_vector2i(v).is_equal(...)` | `assert_that(v).is_equal(...)` |

### 測試環境的 `_ambient_label` 問題

`_ambient_label` 在 `_ready()` 裡建立，但測試直接 `.new()` 不走 scene tree，導致是 null。解法是在測試裡手動 attach：

```gdscript
func _attach_label(level: AsciiLevelBase) -> Label:
    var lbl := auto_free(Label.new()) as Label
    level._ambient_label = lbl
    return lbl
```

### reports/ 不應該進 git

GDUnit4 每次跑測試會產生 `reports/report_N/` HTML 報告。這些是暫存產物，下次可以加進 `.gitignore`。
