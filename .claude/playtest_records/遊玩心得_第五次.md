## 修正：E 鍵在走廊盡頭會「跳出遊戲」

第四次遊玩發現的問題。玩家站在走廊盡頭（靠近 NPC 或門口），如果進場面板還開著：

- **舊行為**：按 E → 只關面板（還需要再按一次 E 才觸發 NPC）
- 玩家以為按一次 E 應該會「過去」，結果第二次 E 才跳場景，感覺像突然跳出

**修正方式**：在 `AsciiLevelBase._input()` 裡，面板關閉後立刻檢查四周是否有 NPC，有的話同一次 E 鍵就觸發互動：

```gdscript
if _scene_panel_open:
    _close_scene_panel()
    get_viewport().set_input_as_handled()
    if not _in_dialogue:
        for dir in [Vector2i(1,0), Vector2i(-1,0), Vector2i(0,1), Vector2i(0,-1)]:
            if (_player_pos + dir) in _npcs:
                _try_interact()
                break
    return
```

同時移除了空白處按 E 會重開進場面板的行為（原本 `elif not _try_interact_object(): _open_scene_panel()`），現在沒有 NPC 也沒有物件時按 E 不做任何事。

## 修正：UI 色彩——深底淺字

第四次遊玩發現色彩難以閱讀，特別是人物標題。原因：進場面板和對話面板都用了**淺底（tan 暖色）配深字**，和遊戲的黑暗基調不符，對比度也不夠。

修正後全部改為**深底淺字**，與 CLAUDE.md 的 Color Palette（Dark Earth）一致：

| 元素 | 舊 | 新 |
|---|---|---|
| 進場面板底色 | `Color(0.72, 0.65, 0.52)` 淺棕 | `Color(0.10, 0.078, 0.060)` 近黑 |
| 進場面板邊框 | `Color(0.18, 0.20, 0.12)` 深綠 | `Color(0.50, 0.36, 0.14)` 暖金 |
| 進場面板文字 | `Color(0.14, 0.12, 0.09)` 暗褐 | `Color(0.88, 0.84, 0.74)` 米白 |
| 對話面板底色 | `Color(0.75, 0.68, 0.54)` tan | `Color(0.10, 0.078, 0.060)` 近黑 |
| 對話本文顏色 | `Color(0.14, 0.12, 0.09)` 暗褐 | `Color(0.88, 0.84, 0.74)` 米白 |
| 人物名稱顏色 | `Color(0.85, 0.82, 0.72)` 米色 | `Color(0.96, 0.80, 0.38)` 暖金 |

## 修正：字體太小——統一基準為 64px

第四次遊玩發現字體太小，閱讀困難。以開始頁面「Sam Pierre」標題（`64px` 絕對大小）為基準，其餘等比例放大，**全部改為絕對值**（不再依地圖字體縮放）：

| 元素 | 舊（縮放比例） | 新（絕對值） |
|---|---|---|
| 進場面板主文 | `20 * font_size / 28` ≈ 20px | **64px** |
| 進場面板提示 | `13 * font_size / 28` ≈ 13px | **22px** |
| 對話人物名稱 | `13 * font_size / 28` ≈ 13px | **32px** |
| 對話本文 | `16 * font_size / 28` ≈ 16px | **40px** |
| 選項按鈕 | `14 * font_size / 28` ≈ 14px | **32px** |
| 選項標題 | `12 * font_size / 28` ≈ 12px | **26px** |

## GDUnit4 自動化測試

安裝 GDUnit4 v6.1.3（`addons/gdUnit4/`），建立 `tests/test_object_interaction.gd`，**21 個測試全部 PASS**。

涵蓋範圍：

| 類別 | 數量 | 內容 |
|---|---|---|
| `StoryLoader.load_objects()` 解析 | 7 | 四個 level key 存在、各四個物件、座標格式、文字非空 |
| `_get_level_id()` 正確性 | 4 | 四個 level 各自回傳正確 id |
| `_try_interact_object()` 回傳值 | 6 | 四方向 true + 距離太遠 false + 無物件 false |
| 物件座標在地圖範圍內 | 4 | 16 個物件座標全在 map 邊界內 |

執行指令（需先跑一次 `--import` 建 class cache）：

```
godot --headless --path . -s res://addons/gdUnit4/bin/GdUnitCmdTool.gd -a res://tests/test_object_interaction.gd --ignoreHeadlessMode
```

## 打字機效果（Typewriter）

進場面板文字現在逐字顯示（`visible_characters` tween），速度 `0.035` 秒/字。

E 鍵三段邏輯：

| 狀態 | E 鍵效果 |
|---|---|
| 打字機播放中 | 跳過→立即顯示全文 |
| 打字機結束，面板開著 | 關閉面板（若旁邊有 NPC 同時觸發） |
| 面板關著 + NPC 旁邊 | 觸發對話 |
| 面板關著 + 物件旁邊 | 顯示物件文字 |
| 面板關著 + 什麼都沒有 | 無動作 |

## 技術備忘

### StoryLoader.gd

新增獨立的 `StoryLoader.gd`（非 autoload），解析 `story/objects.md` Markdown 格式的物件資料。物件座標和文字以 Markdown 存儲，比 GDScript 硬編碼更易由 Sam 直接編輯。

### `--ignoreHeadlessMode`

GDUnit4 v6.1.3 預設拒絕 headless 執行（因為 InputEvent 在 headless 模式不可用）。純邏輯測試加上此 flag 可以安全執行。
