## 動畫系統

ch1 的動作段落現在打字速度跟著劇情節奏變：

| 行 | 事件 | speed |
|---|---|---|
| #17 老鼠抓住皮耶爾 | fast | |
| #18「滾開。」 | slow | |

「滾開。」兩個字慢慢打出來的感覺，比什麼都重。

---

### 畫面震動 / 閃白閃黑 / Glitch 暴衝

行欄位新增 `fx` 陣列，可複數疊加：

```gdscript
"fx": ["shake", "flash_white"]     # 刺中那刻
"fx": ["glitch_spike"]              # 死前獨白
"fx": ["flash_black"]               # 意識消失
```

- **shake**：`map_display` 每幀隨機偏移，`0.35s` 後歸位
- **flash_white / flash_black**：全屏 `ColorRect` tween alpha 0→0.85→0
- **glitch_spike**：臨時把 `stress_level` 拉到 `1.0`，計時結束後還原

刺中那一幀，閃白 + 震動同時觸發，效果比預期暴力。很好。

## 火車死亡動畫

走進地鐵站鐵軌（row 16，`_` 字符區）會觸發一段死亡演出：

1. 文字浮現：「隆隆聲——」
2. ASCII 火車 `══════════╗▶` 從右側以 Tween 衝入
3. 衝到玩家位置：震動 intensity 22 + 閃白
4. 文字更新：「——末班車從黑暗中衝出——」
5. 閃黑 2 秒，回到開始畫面

沒有對話，沒有選擇，就是死掉。比 game over 畫面更有詩意。

## 環境文字系統（Ambient Text）

NPC 鄰近偵測：玩家走到 NPC 旁邊一格，左下角出現淡色文字。

```
ambient_a  ← 對話前
ambient_b  ← 對話結束後
```

四個場景全部配置完畢：

| 場景 | NPC | a（對話前） | b（對話後） |
|---|---|---|---|
| 地鐵站 | 老鼠 `?` | 月台那端有個人。深夜，末班車前。 | 末班車不再回來了。 |
| 警察局 | 李先生 `L` | 李先生正看著這邊。那個油膩的笑。 | 文件永遠是處理不完的。 |
| 餐館 | 某甲 `甲` | 角落那個人。一直沒動。 | 某甲說的那些話，還在耳邊轉。 |
| 各門口 | `>` `<` | 出口文字 | — |

作者改正: 這個的意思是 有些章節的小說是 先描述一個場景 例如車站，然後再接腳色對話 所以@進場景後 可以先開始小說對場景的描述 等觸發對話後 再接腳色對話 。

## 技術備忘

### `var _ambient_label: Label = null` 在 Godot 4.6 會造成 parser error

改成 `var _ambient_label: Node = null` 才能過。原因不明，可能是 Godot 4.6 對 Label 型別在 class 層級宣告有 bug。存取時用 `as Label` cast 解決。
