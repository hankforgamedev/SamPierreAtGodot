## 第四次重點：世界有了文字，文字有了入口

這次從「有靈魂的東西」推進到「像在讀一本你可以走進去的小說」。主要動了兩塊：**進場景面板**讓每個場景有了開場白，**物件互動系統**讓世界的每個角落都有皮耶爾的眼睛。

## 修正：ambient_a 的定義

第三次心得的作者改正：ambient_a 不是「靠近 NPC 才出現的文字」，而是「進場景後的小說式場景描述」。

原本的流程是走到 NPC 旁邊才看到描述，但小說的節奏應該是：

```
進場景 → 先讀場景描述（敘事者） → 探索 → 靠近 NPC → 角色對話
```

這個修正決定了整個第四次的開發方向。

## 進場景面板（Scene Intro Panel）

### 設計

進入任何場景後，全屏敘事面板自動彈出，顯示場景描述文字。玩家無法移動，讀完按 E 關閉。場景內任何時候沒有靠近 NPC 或物件時，按 E 可以重新打開。

E 鍵統一邏輯：

| 狀態 | E 鍵效果 |
|---|---|
| 面板開著 | 關閉面板 |
| 面板關著 + NPC 旁邊 | 觸發對話 |
| 面板關著 + 物件旁邊 | 顯示物件短評 |
| 面板關著 + 什麼都沒有 | 重開進場面板 |

### 實作

在 `AsciiLevelBase.gd` 新增：

- `_setup_scene_panel()` — 建立全屏 Panel（佔螢幕 70%，深色底、暖色邊框）
- `_show_panel_text(text: String)` — 更新 Label 文字並開啟面板（場景面板和物件互動共用）
- `_panel_text_label: Node` — 存 Label 引用，用 `as Label` cast（避免 Godot 4.6 class 層級 Label 宣告 bug）
- `_scene_panel_open: bool` — 面板狀態，開著時鎖移動

各 level 腳本新增 `_get_scene_intro() -> String` override：

```
地鐵站：月台沒有人。深夜，末班車前。隧道那頭是什麼，沒有人說得清楚…
警察局：警察局三樓。辦公室。日光燈的嗡嗡聲從沒停過…
老蕭餐館：老蕭餐館。深夜。廚房早就收了，燈還開著…
中環路：中環路。深夜。街上沒有人…
```

## 物件互動系統（Object Interaction）

### 設計

Disco Elysium 式。地圖上的現有符號（`%` 垃圾堆、`[DESK]`、`[TABLE]`、`[COUNTER]`、積水）都可以互動。玩家走到旁邊按 E，出現皮耶爾的短評或黑色笑話。

不需要 NPC，不需要對話 chapter。就是世界的質感。

### 實作

新增 `scripts/ObjectData.gd`（autoload，在 DialogueData 之後）：

```gdscript
var OBJECTS: Dictionary = {
    "station": { Vector2i(14,11): {"text": "垃圾。或者說是歷史…"}, … },
    "office":  { Vector2i(3,7):   {"text": "桌上有三份文件…"}, … },
    "restaurant": { … },
    "street":  { … },
}
```

四個場景共 **16 個**可互動物件。

在 `AsciiLevelBase.gd` 新增：

- `_get_level_id() -> String` — 虛方法，各 level override 回傳 `"station"` / `"office"` / `"restaurant"` / `"street"`
- `_objects: Dictionary` — `_ready()` 從 `ObjectData.OBJECTS.get(_get_level_id(), {})` 載入
- `_try_interact_object() -> bool` — 四方向掃描 `_objects`，找到就 `_show_panel_text()`，回傳 bool

### E 鍵修正細節

`_try_interact_object()` 回傳 bool 很重要——E 鍵邏輯用 `elif not _try_interact_object()` 才不會在沒有物件時意外重開進場面板：

```gdscript
if near_npc:
    _try_interact()
elif not _try_interact_object():   # false = 沒有物件，才 fallthrough
    _open_scene_panel()
```

## 技術備忘

### `_panel_text_label: Node = null`

同 ambient label 的問題——Godot 4.6 在 class 層級宣告 `Label` 型別會 parser error。改用 `Node`，存取時 `as Label` cast。

### `Vector2i` 作為 Dictionary key

GDScript 4 支援 `Vector2i` 當 dict key，但必須用 `Vector2i(x, y)` literal，不能用 `Vector2`（float），否則執行時 key 查找靜默回傳 null。
